# Pre-Class Warm-Up (~15 minutes)

## When to use
The learner is reviewing shortly before a live class ("class starts soon", "warm me up for
module 2", "quick refresher before today", "remind me what we covered").

## Steps

1. **Find the module.** Determine which module the class is about (ask if unclear, or use the
   next module from `progress/learner-progress.md`).
2. **Open the bridge file.** Read `recap-and-preview.md` in that module's folder. If it's
   thin or missing, fall back to the *previous* module's `lesson.md` summary for the recap
   and *this* module's objectives for the preview.
3. **Recap interactively — don't just read it out.**
   - Surface the prior module's key takeaways as quick prompts: "Last time we covered X —
     can you tell me in a sentence why it mattered?"
   - Give a light nudge if they blank; this is warm-up, not a quiz, so keep it low-pressure.
4. **Bridge.** In a line, connect the previous module to today's.
5. **Preview today.** Share what they'll be able to do and flag the one trickiest concept so
   they walk in primed for it.
6. **Land the one thing.** End on the single most important idea for today's class.
7. **Keep it to ~15 minutes.** Tight and energizing, not a full lesson. Don't start teaching
   the new material in depth — just prime them.

## Don't
- Don't re-teach the whole prior module — this is a refresher.
- Don't turn it into a graded quiz; keep the tone light and confidence-building.
- Don't run long. If they want more, suggest the full `teach-module` after class.
